# 豆包（Doubao）配置指南

## ⚠️ 重要提示

豆包（火山引擎 Doubao）**不支持通用的模型名称**，而是需要你在火山引擎控制台创建的特定**工作负载 ID（Endpoint ID）**。

---

## 快速开始（3 步）

### 步骤 1️⃣ 获取工作负载 ID

1. 登录 [火山引擎控制台](https://console.volcengine.com/)
2. 导航到 **文本生成** 或 **大模型推理** 服务
3. 创建或找到现有的工作负载
4. 复制工作负载 ID，格式如：`ep-20250101xxx-xxxxx`

### 步骤 2️⃣ 在配置中心添加豆包

1. 打开 Skill Evaluator
2. 进入 **配置中心** 选项卡
3. 点击 **+ 添加新模型配置**
4. 选择：**字节跳动 / 火山引擎 (Doubao)**

### 步骤 3️⃣ 填写配置信息

| 字段 | 值 |
|------|-----|
| **显示名称** | 你的工作负载名称（如 `My Doubao Pro`） |
| **模型** | 粘贴工作负载 ID（如 `ep-20250101xxx-xxxxx`） |
| **API Base URL** | `https://ark.cn-beijing.volces.com/api/v3` |
| **API Key** | 你的火山引擎 API Key |

### 步骤 4️⃣ 保存并测试

1. 点击 **保存**
2. 在技能评估器中选择新添加的配置
3. 运行评估测试

## 详细配置步骤

### 1. 获取 API Key

#### 方式 A：使用 API 密钥
```
1. 进入 https://console.volcengine.com/
2. 左侧菜单 → 账户与权限 → API 密钥管理
3. 创建新密钥或使用现有密钥
4. 复制访问密钥（Access Key）
```

#### 方式 B：使用 Token
```
1. 进入控制台首页
2. 右上角 → 安全管理 → Token 管理
3. 创建新 Token
4. 复制 Token 值
```

### 2. 获取工作负载 ID

```
火山引擎控制台 → 文本生成服务 → 在线推理
└─ 找到你的工作负载
   └─ 复制 Endpoint ID
      └─ 格式：ep-xxxxxxxxxx
```

### 3. 配置验证

添加后，系统会自动验证配置。验证成功的标志：
- ✅ 显示绿色的"已配置"状态
- ✅ 在技能评估器中能正常选择
- ✅ 可以成功运行评估

## 常见问题

### ❓ Q: 为什么显示"模型不存在"？

**A:** 可能原因：
1. ❌ 工作负载 ID 输入有误（应该是 `ep-` 开头）
2. ❌ 工作负载已删除或停用
3. ❌ API Key 权限不足

**解决**：
- 检查工作负载 ID 是否正确
- 确认工作负载在火山引擎控制台是可用的
- 检查 API Key 是否有权访问该工作负载

### ❓ Q: 如何找到我的工作负载 ID？

**A:** 
```
火山引擎控制台
├─ 选择产品：文本生成
├─ 左侧菜单：在线推理
└─ 找到你创建的工作负载
   └─ 点击进去
   └─ 详情页面会显示：Endpoint ID
```

### ❓ Q: 可以使用多个工作负载吗？

**A:** 可以！在配置中心添加多个豆包配置，每个对应一个不同的工作负载 ID。

### ❓ Q: API Key 应该填写什么？

**A:** 填写你的**火山引擎 API 密钥**，不是其他云服务的密钥。

## 工作负载 ID 格式

豆包的工作负载 ID 通常格式为：
```
ep-yyyymmddxxx-xxxxx
```

**示例**：
```
✅ 正确格式
ep-20250115abc-12345
ep-20250114xyz-67890

❌ 错误格式
doubao-pro-32k          ← 这是通用模型名（不支持）
my-custom-model         ← 自定义名称（不支持）
kimi-k2-5               ← 其他供应商的模型
```

## 故障排除

### 场景 1: 404 错误 - "模型不存在"

```
错误：The model or endpoint ep-xxx-xxx does not exist
```

**检查清单**：
- [ ] 工作负载 ID 是否正确复制？
- [ ] 工作负载是否已启用？（在火山引擎控制台检查）
- [ ] 是否有该工作负载的访问权限？

### 场景 2: 403 错误 - "权限不足"

```
错误：Access denied or invalid credentials
```

**检查清单**：
- [ ] API Key 是否正确？
- [ ] API Key 是否有过期？
- [ ] API Key 的权限是否包括该工作负载？

### 场景 3: 连接超时

```
错误：Connection timeout
```

**检查清单**：
- [ ] 网络连接是否正常？
- [ ] 火山引擎服务是否在线？
- [ ] 工作负载是否处于正常运行状态？
