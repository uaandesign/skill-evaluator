### 2️⃣ Judge 模型失败时的优雅降级

**问题**：Judge 模型经常连接失败（特别是 Qwen），导致整个评估失败

**解决**：Judge 失败时，自动跳过使用执行结果生成基础评估

#### 工作原理

```
评估开始
  ↓
Phase 1: 执行所有测试用例 (使用选择的模型)
  ↓
Phase 2: Judge 评分
  ├─ 成功? → 显示完整评分 ✅
  └─ 失败? → 自动降级 ⚠️
           ↓
        基于通过率生成评分
        显示黄色警告
```

#### 评分规则

| 通过率 | 评分 | 情况 |
|--------|------|------|
| ≥ 80% | 70 分 | 优秀 |
| 60-80% | 50 分 | 良好 |
| < 60% | 30 分 | 需改进 |

#### 前端表现

**黄色警告提示**：
```
⚠️ Judge 模型不可用
Judge 模型连接失败，本次评估仅基于执行结果生成。
评分为估计值，不代表完整评估。
```

**指示灯颜色**：
- 🟢 绿色 = 完整执行评估（Judge 正常）
- 🟡 黄色 = 部分执行评估（Judge 失败但已降级）

## ❓ FAQ

**Q: Judge 被跳过后评分准确吗？**
A: 通过率计算准确，但维度评分是估计值。建议配置 Claude 备用 Judge。

**Q: 如何避免 Judge 跳过？**
A: 
1. 配置 `ANTHROPIC_API_KEY` (Claude 备用方案)
2. 使用 OpenAI 代替 Qwen（更稳定）
3. 检查网络连接

**Q: 导出报告会显示吗？**
A: 是的，报告会标记 `judge_skipped: true` 和失败原因。
