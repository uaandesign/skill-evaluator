# 专项评估体系实现文档

## 概述

基于《Skill评估平台 - 专项评估体系（Claude适配版·聚焦核心类别）》文档，已完成专项评估体系的V1.0核心功能实现。该体系为4类核心Skill（文本生成、代码生成、数据采集、竞品调研）提供专属评估，与通用评估结合形成：

**总体评分 = 通用评估(60%) + 专项评估(40%)**

---

## 已实现功能

### 1. 技能分类与标签定义

**文件**: `src/specializedRules.js`, `src/components/SkillCategorySelector.jsx`

✅ 预设4类核心Skill标签：
- **文本生成类** (text-generation)：文档优化、文案生成、需求梳理
- **代码生成类** (code-generation)：组件代码、前端页面生成
- **数据采集类** (data-collection)：设计数据、用户行为、业务数据采集
- **竞品调研类** (competitor-research)：竞品设计、功能、行业趋势调研

✅ 技能上传时增加类别选择器
- 用户在技能库上传Skill时，可选择一个类别
- 未选择则仅执行通用评估
- 已选择则在"技能评估"标签页显示专项维度

### 2. 专项评估规则与维度定义

**文件**: `src/specializedRules.js`

每类Skill均包含4个专项维度，每维10分（总40分）：

#### 文本生成类
- **可读性提升** (10分)：优化后可读性≥15%、冗余降低≥20%
- **专业度匹配** (10分)：术语准确率100%、符合文档规范
- **信息完整性** (10分)：核心信息保留率100%、无篡改
- **格式规范性** (10分)：Markdown格式合规、结构清晰

#### 代码生成类
- **语法正确性** (10分)：语法合规率100%、无乱码
- **可运行性** (10分)：代码可编译运行、无报错
- **规范符合性** (10分)：符合前端规范、设计token
- **性能与安全** (10分)：无性能隐患、无安全漏洞

#### 数据采集类
- **数据准确性** (10分)：准确率100%、无遗漏
- **采集效率** (10分)：耗时达标、批量成功率≥95%
- **合规性** (10分)：符合法规、无敏感信息采集
- **可用性** (10分)：可直接用于分析、无冗余

#### 竞品调研类
- **信息全面性** (10分)：覆盖核心调研维度、无遗漏
- **信息准确性** (10分)：准确率100%、信息最新
- **分析深度** (10分)：有竞品对比、设计建议
- **格式规范性** (10分)：结构清晰、可直接参考

每维度包含详细评分rubric（1-5分制），确保评分一致性。

### 3. 专项测试用例生成

**文件**: `src/specializedRules.js`

✅ 为每类Skill生成3-4条专属测试用例模板：
- **正常场景**: 基础功能验证
- **专业场景**: 领域特定验证
- **边界情况/安全场景**: 异常处理验证

用户可在测试前预加载或自定义这些模板。

### 4. 三阶段评估流程（扩展通用评估）

**文件**: `server.js` 中 `/api/evaluate-skill` 端点

#### Phase 1: 执行（Execute）
- 用SKILL.md作为系统提示，逐条执行测试用例
- 收集实际输出和执行耗时

#### Phase 2: 判分（Judge - 通用评估）
- Judge LLM基于4个通用维度评分（有用性/稳定性/准确性/安全性）
- 输出通用评分（1-5分制）

#### Phase 3: 专项评估（Specialized - 新增）
- 仅当skill_category非空时执行
- 专项评估LLM基于类别特定维度评分（4个维度×1-5分）
- 生成专项弱点分析和优化建议

### 5. 加权评分合并

**文件**: `server.js`

✅ 评分合并逻辑：
```
如果有专项评分:
  总体评分 = 通用评分 × 60% + 专项评分 × 40%

如果无专项评分:
  总体评分 = 通用评分 × 100%
```

响应中包含：
- `summary.generic_score`: 通用评分
- `summary.specialized_score`: 专项评分
- `summary.overall_score`: 总体评分（加权后）

### 6. 结果展示与导出

**文件**: `src/components/SkillEvaluatorModule.jsx`

✅ 评估结果展示：
- 新增"专项评估维度"板块，展示4个维度的1-5分评分
- 标注占比权重（40%）
- 通用与专项弱点分析合并展示
- 优化建议包含来源标记（通用/专项）

✅ 报告导出（MD格式）：
- 包含专项评分信息
- 包含专项维度详情
- 包含专项优化建议

### 7. 代码沙箱（前端代码运行验证）

**文件**: `src/components/CodeSandbox.jsx`

✅ 简单的前端沙箱环境：
- 支持HTML/CSS/JS代码隔离执行

- 用于代码生成类Skill的可运行性验证

### 8. 专项规则辅助工具

## 使用流程

### 用户流程

1. **上传技能**
   - 前往"技能库" → "上传技能"
   - 填写名称、描述、上传文件
   - **选择技能类别**（新增）
   - 确认上传

2. **评估技能**
   - 前往"技能评估"
   - 选择模型、技能、版本
   - 生成或上传测试用例
   - 点击"测试"

3. **查看评估结果**
   - **技能评估**标签页：
     - 综合评分（60% 通用 + 40% 专项）
     - 评分构成（质量/功能/安全 + 专项维度）
   - **测试结果**标签页：详细用例结果
   - **优化方案**标签页：通用+专项建议（合并排序）

4. **导出报告**
   - 点击"导出报告 (MD)"
   - 下载包含专项信息的完整评估报告

## 数据流

```
技能上传
  ↓
选择类别 (category)
  ↓
技能评估请求
  ↓
Phase 1: 执行 (实际输出)
  ↓
Phase 2: Judge评估 (通用维度)
  ↓
Phase 3: 专项评估 (仅当category非空)
  ↓
评分合并
  ├─ 通用评分 × 60%
  └─ 专项评分 × 40%
  ↓
总体评分 + 合并建议
  ↓
UI展示 + 报告导出
```

## 注意事项

1. **评分稳定性**：专项评估prompt已按照设计文档格式规范化，但由于LLM的随机性，仍可能出现波动。建议多次评估取平均值。

2. **测试用例质量**：自动生成的模板为基础，用户应根据业务场景补充专业测试用例。

3. **沙箱隔离**：CodeSandbox使用iframe隔离，仅支持标准HTML/CSS/JS，不支持Node.js或外部库。

4. **Category可选**：未选择类别的Skill仅执行通用评估，性能最优。

## 关键算法

### 加权评分计算

```javascript
// Phase 2: 通用评分（1-5分 → 0-100分）
const mapTo100 = (s) => (s ?? 3) * 20;

// Phase 3: 专项评分（同上）
const specializedScore = avg(dimensionalScores) * 20;

// 最终总分
const finalScore = genericScore * 0.6 + specializedScore * 0.4;
```

### 优化建议合并

```javascript
// 通用建议 + 专项建议 → 按优先级排序
const merged = [...generic, ...specialized];
merged.sort((a, b) => priorityOrder[a.priority] - priorityOrder[b.priority]);
```
