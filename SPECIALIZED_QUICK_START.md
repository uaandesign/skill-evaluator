# 专项评估快速开始指南

## 5分钟快速开始

### 步骤1: 上传Skill并选择类别

1. 前往「**技能库**」
2. 点击「**上传技能**」
3. 填写基本信息
4. **在"技能类别"部分选择一个类别**：
   - 📝 **文本生成类** - 用于文档优化、文案生成
   - 💻 **代码生成类** - 用于HTML/CSS/React等代码生成
   - 📊 **数据采集类** - 用于数据收集和整理
   - 🔍 **竞品调研类** - 用于竞品和行业分析
5. 确认上传

### 步骤2: 创建测试用例

1. 前往「**技能评估**」
2. 选择刚上传的技能和版本
3. 点击「**智能生成测试用例**」
   - 系统会根据选择的类别**自动生成对应类型的测试用例**
   - 如文本生成类会生成：可读性测试、术语准确性测试等
   - 代码生成类会生成：语法检查、可运行性测试等
4. 或者上传自己编写的测试用例JSON

### 步骤3: 执行评估

1. 确保已选择：
   - ✅ 大模型
   - ✅ 技能
   - ✅ 测试用例
2. 点击「**测试**」按钮
3. 等待评估完成（通常 30-60秒）

系统会自动执行：
- **Phase 1**: 大模型基于SKILL.md逐条执行测试用例
- **Phase 2**: Judge模型评分（通用4维度）
- **Phase 3**: 专项评估模型评分（类别特定4维度）

### 步骤4: 查看结果

#### 技能评估标签
- **综合评分**：100分制，已包含通用(60%) + 专项(40%)加权
- **评分构成**：展示通用的3个维度 + 专项的4个维度

#### 专项评估维度（新增）
显示该类别特有的4个维度评分：

**文本生成类**
- 可读性提升：文本是否更简洁清晰
- 专业度匹配：术语是否准确规范
- 信息完整性：核心信息是否保留
- 格式规范性：Markdown格式是否规范

**代码生成类**
- 语法正确性：代码是否无错误
- 可运行性：代码是否能执行
- 规范符合性：是否遵循编码规范
- 性能与安全：是否有性能/安全问题

**数据采集类**
- 数据准确性：数据是否准确完整
- 采集效率：采集速度和成功率
- 合规性：是否符合法规和隐私规范
- 可用性：数据是否易于使用

**竞品调研类**
- 信息全面性：调研维度是否完整
- 信息准确性：信息是否准确和最新
- 分析深度：分析是否深入有见地
- 格式规范性：文档结构是否清晰

#### 优化方案标签
- 包含**通用建议** + **专项建议**（合并后按优先级排序）
- 点击「**一键优化**」自动改进Skill
- 生成新版本（v1.1, v1.2, ...）

### 步骤5: 导出报告

1. 在「**技能评估**」标签页右上角
2. 点击「**导出报告 (MD)**」
3. 自动下载包含以下内容的Markdown文件：
   - 综合评分和评分构成
   - 通用维度详情
   - **专项维度详情**（新增）
   - 完整测试执行过程
   - 弱点分析和优化建议

## 核心差异：通用 vs 专项

| 维度 | 通用评估 | 专项评估 |
|-----|--------|--------|
| **适用范围** | 所有Skill必测 | 仅4类核心Skill（可选） |
| **评分维度** | 4个通用维度 | 4个类别特定维度 |
| **评分权重** | 60% | 40% |
| **评估方式** | 通用质量标准 | 领域特定标准 |
| **优化聚焦** | 基础质量 | 业务价值 |

## 常见问题

### Q: 如果我不选择类别会怎样？
**A**: 只执行通用评估（Phase 1-2），不执行专项评估（Phase 3）。总分 = 通用评分×100%。这对于通用Skill或prototype验证很有用。

### Q: 我可以修改专项维度吗？
**A**: V1.0不支持（设计固定），V1.1计划添加。如需自定义维度，可联系开发团队。

### Q: 专项评估要多久？
**A**: 额外耗时 ~1-2秒（LLM调用），总耗时仍在 <90秒内。

### Q: 一键优化后如何验证效果？
**A**:
1. 优化后自动生成新版本（v1.1 → v1.2）
2. 前往「技能评估」选择新版本
3. 点击「测试」重新评估
4. 对比新旧版本分数

### Q: 如何理解评分权重？
**A**:
- **通用评估(60%)**: 保证Skill的基础质量（稳定、安全、可用）
- **专项评估(40%)**: 突出Skill的业务价值（效果、效率、专业性）
- 两者都重要，任一低分都会影响总分

### Q: 导出的报告可以做什么？
**A**:
- 分享给团队进行代码审查
- 作为优化记录和版本对比
- 提交到文档系统进行归档
- 用于持续监测Skill质量

## 最佳实践

### ✅ 做这些

1. **为Skill选择合适的类别**
   - 准确的类别 → 更精准的专项评估 → 更有针对性的优化建议

2. **编写高质量的测试用例**
   - 包括边界情况和异常场景
   - 与实际使用场景贴近

3. **循环迭代**
   ```
   评估 → 查看建议 → 一键优化 → 重新评估 → 对比改进
   ```

4. **定期复测重要Skill**
   - 跟踪质量趋势
   - 早期发现问题

### ❌ 避免这些

1. **盲目选择类别**
   - 应根据Skill的实际功能选择，误选会导致评估不准确

2. **忽视专项维度评分**
   - 专项维度反映Skill的核心价值，应优先关注

3. **一次性优化后就放弃**
   - 一键优化只是起点，建议继续细化和迭代

4. **不测试优化效果**
   - 优化后务必重新评估，验证改进有效性
