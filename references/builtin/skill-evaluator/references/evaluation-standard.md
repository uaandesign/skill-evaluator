# 通用 Skill 评估标准

Rubric version：`generic-skill-rubric-v1`

本文件面向人类评审者，用于解释评估标准、分值和规则背景。平台 agent 的主读取入口是同目录上级的 `SKILL.md`。

本标准定义平台层面的通用 skill 测试。它不评估领域专项正确性、业务规则正确性或火山场景要求，也不再评估平台集成项。专项测试和火山测试应作为独立叠加层使用本报告，但不要改变本通用分数。

## 稳定性要求

同一份 skill 内容必须在每次评估中得到同一评分。评估器遵守以下规则：

- 只对 skill 文件夹执行确定性静态分析。
- 不调用 LLM、外部服务、网络资源、随机函数、系统时间或文件时间戳。
- 分析前统一换行符。
- 哈希和评分前对所有文件路径排序。
- 使用固定整数检查项评分。
- 默认不输出时间戳。
- 使用 SHA-256 `fingerprint` 标识被评估内容。
- 当权重或规则变化时，必须递增 rubric version。

## 分数档位

| 分数 | 档位 | 含义 |
| --- | --- | --- |
| 90-100 | A | 可进入平台通用准入 |
| 80-89 | B | 可用，但存在少量质量缺口 |
| 70-79 | C | 进入平台前需要定向修订 |
| 60-69 | D | 通用就绪度较弱 |
| 0-59 | F | 不适合进入通用平台评估 |

## 评分规则

总分为 100 分。

### 1. 元数据与触发能力：15 分

| ID | 分值 | 确定性检查 |
| --- | ---: | --- |
| M1 | 3 | `SKILL.md` 存在，并以 YAML frontmatter 分隔符开头。 |
| M2 | 3 | Frontmatter 中存在有效的 hyphen-case `name`，且长度不超过 64 个字符。 |
| M3 | 3 | Frontmatter 中存在非空 `description`。 |
| M4 | 3 | Description 包含触发语义，例如 `use when`、`when`、`for`，或中文的“用于”“当”“适用”“使用”。 |
| M5 | 3 | Description 足够具体：长度在 40-1024 字符之间，且不包含尖括号。 |

### 2. 流程可执行性：20 分

| ID | 分值 | 确定性检查 |
| --- | ---: | --- |
| P1 | 3 | 正文有清晰的一级标题。 |
| P2 | 4 | 正文包含流程、步骤或工作流描述。 |
| P3 | 3 | 正文说明输入、目标文件、目标目录或前置条件。 |
| P4 | 3 | 正文说明输出、交付物或完成标准。 |
| P5 | 4 | 正文包含验证、测试或质量门禁说明。 |
| P6 | 3 | 正文包含具体命令、示例或调用方式。 |

### 3. 渐进披露与简洁性：15 分

| ID | 分值 | 确定性检查 |
| --- | ---: | --- |
| D1 | 4 | `SKILL.md` 正文保持简洁：不超过 500 行，近似 token 数不超过 5,000。 |
| D2 | 3 | 当存在 `references/` 文件时，正文链接或提及这些引用资料；没有引用资料时得满分。 |
| D3 | 3 | 当存在 `scripts/` 文件时，正文链接或提及这些脚本；没有脚本时得满分。 |
| D4 | 3 | 不保留真实模板残留，例如待办标记、待定标记、示例占位脚本或模板引导语。 |
| D5 | 2 | skill 文件夹中不存在常见辅助杂物文件，例如 `README.md`、`CHANGELOG.md` 或 `INSTALLATION_GUIDE.md`。 |

### 4. 资源集成：15 分

| ID | 分值 | 确定性检查 |
| --- | ---: | --- |
| R1 | 3 | 资源文件只位于预期目录：`scripts/`、`references/`、`assets/` 或 `agents/`；顶层 license 文件允许存在。 |
| R2 | 3 | 每个 Python 或 shell 脚本都有直接执行入口，例如 shebang、`if __name__ == "__main__"` 或清晰 usage 文本；没有脚本时得满分。 |
| R3 | 3 | 存在脚本时，脚本提供 CLI help 或参数解析；没有脚本时得满分。 |
| R4 | 3 | Markdown 引用文件包含一级标题；没有引用文件时得满分。 |
| R5 | 3 | 资源目录中不保留 init 生成的示例占位文件。 |

### 5. 验证就绪度：20 分

| ID | 分值 | 确定性检查 |
| --- | ---: | --- |
| V1 | 4 | 正文提到验证、测试、检查、评估或评分。 |
| V2 | 4 | 正文定义成功标准、阈值、通过/失败、准入或分数解释。 |
| V3 | 4 | 正文说明如何处理修订、迭代、对比或基线。 |
| V4 | 4 | 正文提到证据产物，例如报告、日志、JSON、diff、截图或输出。 |
| V5 | 4 | 正文提示避免上下文泄露、非确定性、不安全副作用或其他验证完整性风险。 |

### 6. 安全性与运行可靠性：15 分

| ID | 分值 | 确定性检查 |
| --- | ---: | --- |
| S1 | 4 | 没有在缺少安全说明的情况下出现破坏性 shell 模式。 |
| S2 | 4 | 当正文包含命令或脚本时，说明前置条件、权限、环境或依赖。 |
| S3 | 3 | 正文包含确认、批准、沙箱、避免副作用或避免误操作的护栏说明。 |
| S4 | 4 | 不出现明显密钥占位值或凭据处理反模式。 |

## 确定性报告字段

必需机器可读字段：

- `rubric_version`
- `skill_path`
- `fingerprint`
- `score`
- `grade`
- `generic_assessment`
- `category_scores`
- `checks`

可选字段：

- 当提供基线 JSON 报告时，包含 `baseline_comparison`

## 通用准入门禁

平台通用测试使用以下门禁：

- 当总分不低于 90，且所有一级维度得分不低于该维度可得分的 70% 时，通用测试通过，评估结果中的 `generic_assessment.tag` 为 `通过`。
- 当总分为 80-89，或任一一级维度低于 70% 时，给出警告，评估结果中的 `generic_assessment.tag` 为 `警告`。
- 当总分低于 80 时，判定为未达到通用准入要求，评估结果中的 `generic_assessment.tag` 为 `不通过`。

专项测试和火山测试可以使用更严格的门禁，但不应修改本通用分数。
